## Введение
### Концепция проекта
Для специалистов, которым нужно продвигать свою экспертность, данная система TopicHub является Интернет-форумом, который обеспечивает возможность публиковать статьи и проводить ни их основе обсуждения. Система будет предлагать возможность следить за экспертами из различных областей, или хабами. Что позволит заинтересованным узнать другие мнения по данной теме, а также предложить свои идеи. Будучи экспертом в конкретной области, специалист сможет расширить круг знакомств, а также поделиться своим мнение через публикацию статей и оставляя реакцию. 

По используемым функциям пользователи делятся на 3 вида: гость, автор, модератор.

Данные для входа в роли модератора: логин: admin, пароль: 123456

### Установка
Склонировать проект: 
```bash
git clone https://github.com/zheki4sh05/TopicHub-web-forum.git
```

```bash

```

## Функциональные требования

### Для гостя

1. Просмотр статей
   1. Просмотры
   2. Лайки/Дизлайки
   3. Комментарии
   4. Просмотр по хабам
2. Поиск статьи
   1. Поиск по автору
   2. Поиск по рейтингу
   3. Поиск по теме
   4. Поиск по ключевым словам

### Для автора

1. Публикация статей
   1. Оформление статьи
      1. Список
      2. Заголовок
      3. Основной текст
      4. Изображение
   2. Добавление ключевых слов
   3. Отправление на модерацию
   4. Просмотр профиля
2.  Наблюдение за другими авторами
    1. Подписка на автора
    2. Добавление в закладки, убрать
3. Обсуждение
   1. Комментирование (Создание, Удаление, Изменение)
   2. Жалоба
   3. Лайк/дизлайк
4. Авторизация
   1. Создание аккаунта
   2. Управление профилем

### Для модератора

1. Управление статьями (Отклонение, Удаление, Модерация)
2. Управление хабами (Добавление, Удаление, Изменение)
3. Управление авторами (Блокировка, Разблокировка)
4. Управление комментариями (Просмотр жалоб, Удаление комментария)

### Выпуск v3

#### Результаты разработки

Начало миграции на Spring Framework.

В новой версии backend часть переписана на Spring Framework. С помощью данного фреймворка реализовалась авторизация (Spring Security with Basic Authentication), взаимодействие с БД посредством Spring Data JPA, а также внедрение бинов (компонентов приложения). Использование чистого Spring подразумевает ручную настройку некоторых частей приложения. В частности вручную настраивался DispatcherServlet, связь с БД (Persistence Configuration).
Админ-панель переведана на MVC Thymeleaf + Bootstrap, что позволило уменьшить размер клиенткого SPA. С архитектурной точки зрения теперь за счет Spring IoC не нужно управлять зависмостями с помощью фабрик (V1,V2). Теперь все основные компоненты это бины, а Spring отвечает за создание экземпляра, настройку и компонование бинов. Описание API можно найти в package с именем Controllers.


### Выпуск v4

#### Результаты разработки

Теперь приложение полностью переведено на SpringBoot. Данный подход позволил избавить архитектуру от дополнительных конфигурация (аннотация @SpringBootApplication), а также упростило разработку тестов за счет аннотаций @SpringBootTest и @AutoConfigureMockMvc. Безопасноть приложения базируется на основе токена. То есть теперь для проверки аутентификации и авторизации используется access токен, что в вместе с refresh позволяет пользователю вводить личные данные как можно реже. JWT значительно безопаснее. Basic Authentication отправляет учетные данные (имя пользователя и пароль) в каждом запросе в нешифрованном виде. JWT использует закодированный токен, который не раскрывает учетные данные.
Стоит отметить, что клиенсткое приложение на ReactJS тоже теперь работает по JWT. Автоматическая вставка и Refresh токена (ошибка 401) реализованы с помощью Axios Interceptors. Access токены хранится в Redux, а Refresh в localstorage. Хранение токена в LS браузера позволяет пользователю продолжать быть авторизованным в приложении.
